Pasos de explotación

  1. Realiza múltiples intentos de login con contraseñas incorrectas.
  2. Observa que el archivo de log permanece completamente vacío — no se registra ningún evento.
  3. Esto significa que un ataque de fuerza bruta pasaría completamente desapercibido.
  4. En una aplicación segura, cada intento fallido debería generar una entrada de log con: timestamp, IP, username intentado y User-Agent.
  5. Combina este lab con el de Auth Failures: lanza hydra y confirma que no hay registro de los miles de intentos:
hydra -l admin -P /usr/share/wordlists/rockyou.txt \
  localhost -s 5000 http-post-form \
  "/logging/login:username=^USER^&password=^PASS^:incorrectas"

Impacto

Sin detección de intrusos Sin forense Sin alertas

Exploitation Steps

  1. Make multiple login attempts with wrong passwords.
  2. Notice that the log file remains completely empty — no events are ever recorded.
  3. This means a brute-force attack would go completely undetected.
  4. In a secure app, each failed attempt should generate a log entry with: timestamp, IP, attempted username and User-Agent.
  5. Combine with the Auth Failures lab: run hydra and confirm thousands of attempts leave no trace:
hydra -l admin -P /usr/share/wordlists/rockyou.txt \
  localhost -s 5000 http-post-form \
  "/logging/login:username=^USER^&password=^PASS^:incorrectas"
